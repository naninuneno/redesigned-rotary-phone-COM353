Deprecated <table> attributes

Attribute Description Why is it deprecated?
align Indicates how the table must be aligned in regard of the containing document The <table> element should be styled using CSS instead
bgcolor Defines the background colour of the table and its content The <table> element should be styled using CSS instead
border Defines, in pixels, the size of the frame surrounding the table (this table uses it!) The <table> element should be styled using CSS instead
cellpadding Defines the space between the content of a cell and the border of the cell The <table> element should be styled using CSS instead
cellspacing Defines the size, in percentage or pixels, of the space between two cells - horizontally and vertically The <table> element should be styled using CSS instead
frame Defines which side of the frame surrounding the table must be displayed The <table> element should be styled using CSS instead
rules Defines where rules, i.e. lines, should appear in a table The <table> element should be styled using CSS instead
summary Defines an alternative text use to describe the table in user-agents unable to display it. Allows visually impaired people to get the information from it also Can be replace by a <caption> or <details> element, or the table adjusted so that such description is no longer needed (e.g. using <th> and <thead> elements
width Defines the width of the table The <table> element should be styled using CSS instead

Terminology learned:
HTML
HyperText Markup Language - The set of markup symbols or codes placed in a file intended for display on a Web browser page
XHTML
eXtensible HyperText Markup Language - uses the elements and attributes of HTML with the syntax of XML
DHTML
Dynamic HTML is a term for the combination of technologies which can be used to make interactive websites, governing structure (HTML), presentation (CSS) and behaviour (Javascript and the DOM)
HTML5
The newest version of HTML/XHTML - supported by modern browsers & intended to be backwards compatible. Adds new elements and new functionality e.g. native video & audio
Document Type Definition (DTD)
A doctype statement - identifies the version of HTML contained in your document - placed at the top of a web page document file e.g. for HTML5
HTML Head Section
Contains information that describes the Web page document
HTML Body Section
Contains text and elements that display in the Web page document
HTML Lists
3 types of lists in HTML - ordered (numbered), unordered (bulleted) and definition (terms and their definitions) - lists can also be nested
HTML Tables
Tables can be made up of thead, tbody & tfoot tags, which hold tr (table row), th (table heading) and td (table data) tags
GET
The GET request method requests data from a specified resource. They should never be used when dealing with sensitive data
POST
The POST request method submits data to be processed to a specified resource. It requests that a web server accepts and stores the data enclosed in the body of the request message. It is often used when uploading a file or submitting a completed web form
DOM
The Document Object Model (DOM) is a programming interface for HTML and XML documents. It provides a structured representation of the document and it defines a way that the structure can be accessed from programs so that they can change the document structure, style and content
